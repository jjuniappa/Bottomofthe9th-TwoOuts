# 9회말 2아웃

GitHub Pages에서 바로 실행할 수 있는 모바일 투구 게임 프로토타입입니다.

## 실행

1. 이 폴더의 모든 파일과 `assets` 폴더를 GitHub 저장소 루트에 업로드합니다.
2. GitHub 저장소의 **Settings → Pages**로 이동합니다.
3. **Deploy from a branch**, `main`, `/ (root)`를 선택합니다.
4. 로컬에서는 `index.html`을 바로 열어도 실행됩니다.

## 현재 판정 규칙

- 심판 오차 없음
- 야구공이 스트라이크존에 조금이라도 닿으면 스트라이크 판정 대상
- 공 전체가 존 안에 들어오면 안타 확률 100%
- 존에 아주 조금 걸치면 안타 확률 약 10%
- 중간 구간은 공이 존 안에 들어온 면적 비율에 따라 안타 확률이 10%에서 100%까지 증가
- 타격에 실패하면 헛스윙 스트라이크
- 3스트라이크는 승리, 4볼 또는 안타는 패배

## 폴더 구조

```text
index.html
styles.css
game.js
assets/
  background/stadium_bg.png
  balls/baseball.png
```


## v6 에셋 로딩

GitHub Pages의 폴더/경로 문제를 피하기 위해 `assets.js`에도 배경과 야구공이 내장되어 있습니다. `index.html`, `assets.js`, `game.js`, `styles.css`를 저장소 루트에 함께 올리세요.


## v7 변경사항
- 화면에 표시되던 스트라이크존 라인을 제거했습니다. 판정 영역은 내부 좌표로 유지됩니다.
- 야구공이 포수에게 접근할수록 시작 크기의 약 18%까지 작아집니다.
- 안타 판정 시 확률을 표시하지 않고 `안타`만 표시합니다.
